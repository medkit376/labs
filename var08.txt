ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с инструментом сборки фронтенд-приложений «Vite»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Vite — инструмент для сборки и оптимизации фронтенд-приложений, обеспечивающий быструю разработку с мгновенной горячей заменой модулей (HMR) и оптимизированную сборку для продакшена.
1.2. Область применения
Используется разработчиками SPA-приложений для сборки проектов на React, Angular, Vue и других JavaScript-фреймворках.
1.3. Краткая характеристика возможностей
Vite позволяет быстро запускать сервер разработки с мгновенной горячей перезагрузкой, использовать современные JavaScript-модули (ESM) без сборки в режиме разработки, выполнять оптимизированную сборку для продакшена с минификацией и разделением кода, поддерживать TypeScript, JSX, CSS-препроцессоры и плагины для расширения функциональности.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Vite требуется операционная система Ubuntu 20.04 и выше, установленный Node.js версии 16.0 и выше, оперативная память не менее 2 ГБ и свободное место на диске не менее 500 МБ.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы JavaScript, понимать структуру фронтенд-проектов, уметь работать с терминалом и знать базовые команды Node.js (npm, npx).

3. ПОДГОТОВКА К РАБОТЕ
3.1. Проверка установки
Для проверки установки Vite необходимо выполнить команду vite --version. При успешной установке отобразится версия программы.
3.2. Создание нового проекта
Для создания нового проекта Vite используется команда: npm create vite@latest. Затем необходимо ввести имя проекта (например, my-app), выбрать фреймворк (React, Vue, Angular и т.д.) и вариант (JavaScript, TypeScript). После создания проекта перейти в папку проекта: cd my-app и установить зависимости: npm install.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Запуск сервера разработки
Для запуска сервера разработки используется команда npm run dev или vite. Сервер запускается на порту 5173. Открыть браузер и перейти по адресу http://localhost:5173.
4.2. Горячая перезагрузка (HMR)
При изменении кода страница автоматически обновляется без перезагрузки, сохраняя состояние приложения (Hot Module Replacement).
4.3. Сборка для продакшена
Для сборки приложения для продакшена используется команда npm run build. Собранные файлы сохраняются в папку dist/.
4.4. Просмотр собранного приложения
Для просмотра собранного приложения локально используется команда npm run preview или vite preview. Сервер запускается на порту 4173.
4.5. Настройка Vite
Для настройки Vite необходимо создать файл vite.config.js в корне проекта. Базовый файл конфигурации:
export default {
  root: '.',
  build: {
    outDir: 'dist',
    assetsDir: 'assets',
  },
  server: {
    port: 3000,
    open: true,
  },
};
4.6. Использование плагинов
Для использования плагинов необходимо установить плагин через npm и добавить в vite.config.js. Пример с React-плагином:
npm install @vitejs/plugin-react
В vite.config.js:
import react from '@vitejs/plugin-react';
export default {
  plugins: [react()],
};
4.7. Настройка алиасов (alias)
Для настройки алиасов (сокращенных путей) необходимо добавить в vite.config.js:
export default {
  resolve: {
    alias: {
      '@': '/src',
      '@components': '/src/components',
    },
  },
};
4.8. Использование переменных окружения
Для использования переменных окружения необходимо создать файл .env в корне проекта. Переменные должны начинаться с VITE_, например: VITE_API_URL=http://localhost:8000. В коде переменные доступны через import.meta.env.VITE_API_URL.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Command not found: vite необходимо установить Vite глобально или использовать npx vite. При ошибке Port 5173 is already in use необходимо изменить порт в настройках vite.config.js: server: { port: 3000 }. При ошибке Cannot find module необходимо проверить установку зависимостей и выполнить npm install. При ошибке Failed to resolve import необходимо проверить правильность путей импорта. При ошибке Out of memory при сборке необходимо увеличить память Node.js: NODE_OPTIONS="--max-old-space-size=4096" npm run build. При ошибке ESLint/Prettier не работает необходимо убедиться, что плагины установлены и настроены в настройках редактора.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка ESLint
Для настройки ESLint необходимо установить ESLint: npm install -D eslint. Создать файл .eslintrc.js:
module.exports = {
  extends: ['eslint:recommended', 'plugin:react/recommended'],
  rules: {
    'no-unused-vars': 'warn',
    'react/prop-types': 'off',
  },
};
6.2. Настройка Prettier
Для настройки Prettier необходимо установить Prettier: npm install -D prettier. Создать файл .prettierrc:
{
  "semi": true,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5"
}
6.3. Интеграция Vite с VS Code
Для интеграции Vite с VS Code рекомендуется установить расширения: Vite, ESLint, Prettier и соответствующие расширения для фреймворка (React, Vue, Angular).
6.4. Настройка источника данных в разработке
Для настройки прокси к API-серверу в разработке добавить в vite.config.js:
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
}
6.5. Настройка оптимизации сборки
Для настройки оптимизации сборки добавить в vite.config.js:
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        vendor: ['react', 'react-dom'],
      },
    },
  },
}
