ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с сервером приложений «Apache Tomcat»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Apache Tomcat — сервер приложений для развертывания и запуска Java-веб-приложений, реализующий спецификации Jakarta Servlet и JavaServer Pages (JSP).
1.2. Область применения
Используется разработчиками Java для локального развертывания и тестирования веб-приложений в процессе разработки корпоративного Java-ПО в банковском секторе.
1.3. Краткая характеристика возможностей
Apache Tomcat позволяет развертывать веб-приложения в формате WAR, управлять сессиями пользователей, настраивать пул соединений с базами данных, использовать SSL/TLS для защищенных соединений, управлять виртуальными хостами, предоставляет веб-интерфейс для управления приложениями и поддерживает горячее развертывание без перезапуска сервера.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Apache Tomcat требуется операционная система Ubuntu 20.04 и выше, установленный Java (JDK 11 или 17), оперативная память не менее 2 ГБ, процессор архитектуры x86_64 и свободное место на диске не менее 1 ГБ.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы Java-разработки, принципы работы веб-приложений, структуру WAR-файлов, базовые команды Linux и уметь работать с файловой системой.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск Tomcat
Для запуска Tomcat необходимо перейти в папку /opt/apache-tomcat-*/bin/ и выполнить команду ./startup.sh. Для остановки сервера используется команда ./shutdown.sh.
3.2. Проверка работы
После запуска Tomcat открыть браузер и перейти по адресу http://localhost:8080. При успешном запуске отобразится приветственная страница Apache Tomcat.
3.3. Настройка пользователей для Manager
Для доступа к менеджеру приложений необходимо отредактировать файл /opt/apache-tomcat-*/conf/tomcat-users.xml и добавить пользователя с ролью manager-gui:
<user username="admin" password="admin123" roles="manager-gui,admin-gui"/>

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Развертывание приложения через веб-интерфейс
Для развертывания приложения через веб-интерфейс необходимо открыть http://localhost:8080/manager/html. В разделе «Deploy» выбрать файл WAR и нажать «Deploy». Приложение будет развернуто и доступно по пути /application_name.
4.2. Развертывание приложения через папку webapps
Для развертывания через папку webapps необходимо скопировать WAR-файл в директорию /opt/apache-tomcat-*/webapps/. Tomcat автоматически развернет приложение.
4.3. Удаление приложения
Для удаления приложения через веб-интерфейс необходимо в разделе «Applications» найти приложение и нажать кнопку «Undeploy». При удалении через папку webapps необходимо удалить соответствующий WAR-файл и папку с приложением.
4.4. Просмотр логов
Логи Tomcat находятся в директории /opt/apache-tomcat-*/logs/. Основные файлы: catalina.out — основной лог с выводом приложений; localhost.log — логи развертывания; manager.log — логи менеджера.
4.5. Настройка порта
Для изменения порта необходимо отредактировать файл /opt/apache-tomcat-*/conf/server.xml. Найти строку <Connector port="8080" ...> и изменить порт на нужный (например, 8081). После изменения порта необходимо перезапустить Tomcat.
4.6. Настройка JVM параметров
Для изменения параметров JVM необходимо отредактировать файл /opt/apache-tomcat-*/bin/setenv.sh (создать, если отсутствует). Пример: export CATALINA_OPTS="-Xms512m -Xmx2048m -XX:MaxPermSize=256m"
4.7. Настройка SSL (HTTPS)
Для настройки SSL необходимо сгенерировать сертификат: keytool -genkey -alias tomcat -keyalg RSA -keystore /opt/apache-tomcat-*/conf/keystore.jks. Затем в server.xml раскомментировать и настроить Connector для порта 8443.
4.8. Просмотр активных сессий
Для просмотра активных сессий необходимо перейти в менеджер приложений и нажать на приложение. В открывшемся окне отобразится количество активных сессий.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Tomcat не запускается необходимо проверить, что порт 8080 не занят, командой sudo netstat -tlnp | grep 8080. Также проверить логи в /opt/apache-tomcat-*/logs/catalina.out. При ошибке Permission denied при запуске Tomcat необходимо выполнить chmod +x /opt/apache-tomcat-*/bin/*.sh. При ошибке 404 Not Found при доступе к Manager необходимо добавить роль manager-gui в tomcat-users.xml. При ошибке OutOfMemoryError необходимо увеличить память JVM в setenv.sh. При ошибке Address already in use необходимо изменить порт в server.xml. При ошибке 403 Forbidden необходимо проверить права доступа к папке webapps.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Изменение порта
Для изменения порта необходимо отредактировать файл /opt/apache-tomcat-*/conf/server.xml, найти строку <Connector port="8080" ...> и изменить значение на нужное (например, 8081). Затем перезапустить Tomcat.
6.2. Настройка таймаута сессии
Для настройки таймаута сессии необходимо отредактировать файл /opt/apache-tomcat-*/conf/web.xml. Найти строку <session-timeout> и изменить значение (в минутах). По умолчанию 30 минут.
6.3. Настройка размера запроса
Для настройки максимального размера запроса необходимо добавить параметр maxPostSize в Connector в файле server.xml. Пример: <Connector port="8080" maxPostSize="52428800" .../>
6.4. Настройка виртуальных хостов
Для настройки виртуальных хостов необходимо добавить новый Host в файл server.xml. Пример:
<Host name="myapp.company.com" appBase="webapps" unpackWARs="true" autoDeploy="true"/>
6.5. Включение автоматического развертывания
Для включения автоматического развертывания необходимо установить параметр autoDeploy="true" в Host в файле server.xml.
