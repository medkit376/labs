ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с инструментом создания установочных пакетов «NSIS» (Nullsoft Scriptable Install System)
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
NSIS — инструмент для создания установочных пакетов (инсталляторов) для Windows-приложений, позволяющий создавать профессиональные установщики с графическим интерфейсом.
1.2. Область применения
Используется разработчиками кросс-платформенных настольных приложений для создания установочных пакетов под операционную систему Windows.
1.3. Краткая характеристика возможностей
NSIS позволяет создавать установщики с настраиваемым интерфейсом, поддерживает различные языки интерфейса, позволяет добавлять лицензионные соглашения, выбирать компоненты для установки, создавать ярлыки на рабочем столе, добавлять записи в реестр Windows, создавать деинсталляторы и поддерживает скриптовый язык для настройки поведения установщика.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы NSIS требуется операционная система Ubuntu 20.04 и выше, установленный Wine для запуска Windows-приложений, оперативная память не менее 2 ГБ, процессор архитектуры x86_64 и свободное место на диске не менее 1 ГБ.
2.2. Требования к квалификации пользователя
Пользователь должен знать структуру каталогов приложения, основы скриптового языка NSIS, уметь работать с файловой системой и понимать принципы создания установочных пакетов для Windows.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск NSIS
Для запуска NSIS необходимо открыть терминал и выполнить команду wine ~/.wine/drive_c/Program\ Files/NSIS/makensis.exe или использовать графический интерфейс через wine ~/.wine/drive_c/Program\ Files/NSIS/NSIS.exe.
3.2. Создание первого скрипта
Для создания скрипта необходимо создать файл с расширением .nsi и открыть его в текстовом редакторе. Базовый скрипт включает определение имени установщика, выходного файла, страниц установки и файлов для копирования.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Базовый скрипт установщика
Для создания базового установщика необходимо создать файл installer.nsi со следующим содержимым:
Name "MyApp"
OutFile "MyApp_Setup.exe"
InstallDir "$PROGRAMFILES\MyApp"
Page Directory
Page InstFiles
Section
SetOutPath $INSTDIR
File "myapp.exe"
File "README.txt"
CreateShortcut "$DESKTOP\MyApp.lnk" "$INSTDIR\myapp.exe"
SectionEnd

4.2. Добавление лицензионного соглашения
Для добавления лицензионного соглашения необходимо добавить страницу License перед страницей Directory:
Page License
LicenseData "license.txt"
4.3. Добавление выбора компонентов
Для добавления выбора компонентов необходимо использовать компоненты:
Page Components
Section "Основные файлы" SecMain
SetOutPath $INSTDIR
File "myapp.exe"
SectionEnd
Section "Документация" SecDocs
SetOutPath $INSTDIR\docs
File "docs\*.pdf"
SectionEnd

4.4. Добавление записей в реестр
Для добавления записей в реестр Windows необходимо использовать команды WriteRegStr:
WriteRegStr HKLM "SOFTWARE\MyApp" "InstallPath" $INSTDIR
WriteRegStr HKLM "Software\Microsoft\Windows\CurrentVersion\Uninstall\MyApp" "DisplayName" "MyApp"
WriteRegStr HKLM "Software\Microsoft\Windows\CurrentVersion\Uninstall\MyApp" "UninstallString" "$INSTDIR\uninstall.exe"

4.5. Создание деинсталлятора
Для создания деинсталлятора необходимо добавить секцию для его создания:
WriteUninstaller "$INSTDIR\uninstall.exe"
Section "Uninstall"
Delete "$INSTDIR\myapp.exe"
Delete "$INSTDIR\README.txt"
Delete "$DESKTOP\MyApp.lnk"
Delete "$INSTDIR\uninstall.exe"
RMDir "$INSTDIR"
SectionEnd

4.6. Компиляция скрипта
Для компиляции скрипта необходимо выполнить команду: wine ~/.wine/drive_c/Program\ Files/NSIS/makensis.exe installer.nsi. При успешной компиляции будет создан файл MyApp_Setup.exe.
4.7. Использование графического интерфейса (HM NIS Edit)
Для использования графического интерфейса необходимо запустить HM NIS Edit через wine ~/.wine/drive_c/Program\ Files/NSIS/HM\ NIS\ Edit.exe. В графическом интерфейсе доступны мастер создания скрипта, редактор с подсветкой синтаксиса и кнопка для компиляции.
4.8. Добавление языков
Для добавления поддержки русского языка необходимо добавить в скрипт:
LoadLanguageFile "${NSISDIR}\Contrib\Language files\Russian.nlf"

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Access denied при компиляции необходимо проверить права на запись в папку вывода и запускать с соответствующими правами. При ошибке Installer corrupted необходимо перекомпилировать скрипт и убедиться, что все файлы существуют. При ошибке Section not found необходимо проверить правильность имен секций и их закрытие. При ошибке Command not found необходимо убедиться, что команда поддерживается используемой версией NSIS. При ошибке Invalid path необходимо проверить пути к файлам на корректность и наличие кавычек. При ошибке Out of memory необходимо уменьшить количество файлов или увеличить память.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка внешнего вида установщика
Для настройки внешнего вида установщика используются команды:
BrandingText "Мой продукт"
Caption "Установка MyApp"
XPStyle on
Icon "installer.ico"
6.2. Установка фонового изображения
Для установки фонового изображения на страницы установщика:
SetBrandingImage "background.bmp"
6.3. Настройка языка интерфейса
Для настройки языка интерфейса:
LoadLanguageFile "${NSISDIR}\Contrib\Language files\Russian.nlf"
6.4. Установка размера установщика
Для указания размера установщика в заголовке:
VIAddVersionKey "ProductVersion" "1.0.0"
VIAddVersionKey "ProductName" "MyApp"
VIProductVersion "1.0.0.0"
6.5. Добавление файлов с маской
Для добавления файлов с маской:
File "*.exe"
File "*.dll"
File /r "resources\*.*"
6.6. Настройка параметров сжатия
Для настройки параметров сжатия:
SetCompressor /SOLID lzma
SetCompressorDictSize 64
