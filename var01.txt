ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с инструментом тестирования API «Insomnia»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Insomnia предназначена для отправки HTTP-запросов, тестирования RESTful и GraphQL API, автоматизации проверки ответов сервера и отладки веб-приложений.
1.2. Область применения
Используется разработчиками бэкенда и фронтенда для валидации эндпоинтов (endpoints) в процессе разработки высоконагруженных веб-сервисов.
1.3. Краткая характеристика возможностей
Insomnia позволяет создавать и отправлять HTTP-запросы (GET, POST, PUT, DELETE, PATCH), поддерживает REST и GraphQL API, обеспечивает группировку запросов в коллекции, использование переменных окружений, импорт и экспорт запросов, а также автоматическое форматирование JSON-ответов.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Insomnia требуется операционная система Ubuntu 20.04 и выше, оперативная память не менее 4 ГБ, процессор архитектуры x86_64 и свободное место на диске не менее 500 МБ.
2.2. Требования к квалификации пользователя
Пользователь должен знать базовые принципы работы HTTP-протокола, основы RESTful архитектуры и форматы данных JSON и XML.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск программы
Для запуска Insomnia необходимо открыть меню приложений и выбрать иконку Insomnia или выполнить в терминале команду insomnia.
3.2. Создание рабочей области (Workspace)
После запуска Insomnia необходимо нажать кнопку Create в окне приветствия. Затем выбрать тип Design Document или Collection, ввести имя рабочей области (например, «Проект API») и нажать Create.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Создание HTTP-запроса
Для создания нового запроса необходимо нажать кнопку + (New Request) или использовать сочетание клавиш Ctrl+N. Затем ввести имя запроса (например, «Получение пользователей»), выбрать метод (GET, POST, PUT, DELETE) и нажать Create.
4.2. Настройка URL и параметров запроса
В поле URL необходимо ввести адрес, например, http://localhost:8000/api/users. Для добавления параметров запроса нужно перейти на вкладку Query, нажать Add и ввести имя параметра (например, page) и его значение (например, 1). Для добавления заголовков необходимо перейти на вкладку Headers, нажать Add и ввести, например, Content-Type: application/json.
4.3. Отправка тела запроса (Body)
Для отправки данных в теле запроса необходимо перейти на вкладку Body и выбрать формат данных: JSON для отправки JSON-данных, Form URL Encoded для формы или Multipart Form для файлов. После выбора формата необходимо ввести данные, например:
{
    "username": "test_user",
    "email": "test@example.com"
}
4.4. Отправка запроса
Для отправки запроса необходимо нажать кнопку Send (голубая кнопка) или использовать сочетание клавиш Ctrl+Enter.
4.5. Просмотр ответа сервера
В правой панели отображается статус-код ответа (200 OK, 404 Not Found, 500 Internal Server Error), время выполнения в миллисекундах, размер ответа в байтах и тело ответа в формате JSON, XML или HTML.
4.6. Использование окружений (Environments)
Для использования переменных окружений необходимо перейти в Dashboard (значок домика в левом верхнем углу), нажать Manage Environments и создать Base Environment со следующими переменными:
{
    "base_url": "http://localhost:8000",
    "token": "your_api_token_here"
}
В запросе переменные используются следующим образом: {{ base_url }}/api/users подставится http://localhost:8000/api/users, а Authorization: Bearer {{ token }} подставится указанный токен.
4.7. Группировка запросов в коллекции
Для группировки запросов необходимо в левой панели нажать + рядом с названием рабочей области, выбрать New Folder, ввести имя папки (например, «Пользователи») и перетащить запросы в созданную папку.
4.8. Импорт и экспорт данных
Для экспорта данных необходимо нажать на название рабочей области, выбрать Export, выбрать формат Insomnia v4 (JSON) и сохранить файл. Для импорта данных необходимо нажать на название рабочей области, выбрать Import, затем From File и выбрать файл с расширением .json.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Connection Refused необходимо запустить серверное приложение. При ошибке 404 Not Found следует проверить правильность URL эндпоинта. При ошибке 401 Unauthorized необходимо добавить токен в заголовки Authorization. При ошибке 500 Internal Server Error требуется проверить логи сервера. При ошибке SSL Error необходимо отключить проверку SSL в настройках или установить корректный сертификат. При ошибке Request Timeout следует увеличить таймаут в настройках запроса. При ошибке JSON Parse Error необходимо проверить синтаксис JSON в теле запроса.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Смена темы оформления
Для смены темы оформления необходимо нажать Application в верхнем меню, выбрать Preferences → Themes и выбрать тему Dark или Light.
6.2. Настройка горячих клавиш
Для настройки горячих клавиш необходимо нажать Application → Preferences → Keyboard Shortcuts, найти нужную команду, нажать на нее и ввести новую комбинацию клавиш.
6.3. Изменение размера шрифта
Для изменения размера шрифта необходимо нажать View → Zoom In или Zoom Out или использовать сочетания клавиш Ctrl++ и Ctrl+-.
6.4. Настройка формата ответа
Для настройки формата ответа необходимо нажать на шестеренку (Settings) в правом верхнем углу и в разделе Response выбрать Pretty для форматированного JSON, Raw для необработанного ответа или Preview для HTML-предпросмотра.
6.5. Установка таймаута запроса
Для установки таймаута запроса необходимо нажать на шестеренку (Settings) в правом верхнем углу и в разделе Request изменить значение Timeout (по умолчанию 30000 мс).
