ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с системой эмуляции домашней сети «Home Assistant»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Home Assistant — открытая платформа для управления умным домом, позволяющая подключать, управлять и автоматизировать различные устройства (лампы, розетки, датчики, климат-контроль, камеры) через единый интерфейс.
1.2. Область применения
Используется разработчиками IoT-решений и систем умного дома для тестирования взаимодействия устройств, эмуляции домашней сети, создания сценариев автоматизации и интеграции с различными протоколами (Zigbee, Z-Wave, MQTT, Wi-Fi).
1.3. Краткая характеристика возможностей
Home Assistant позволяет подключать устройства через MQTT и другие протоколы, создавать автоматизации (сценарии, правила), управлять устройствами через веб-интерфейс, поддерживать голосовое управление, предоставлять API для интеграции, использовать Zigbee2MQTT для работы с Zigbee-устройствами и обеспечивать безопасное хранение данных с шифрованием.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Home Assistant требуется операционная система Ubuntu 20.04 и выше, процессор архитектуры x86_64, оперативная память не менее 4 ГБ, свободное место на диске не менее 10 ГБ и наличие USB-адаптера Zigbee (Conbee II, CC2531) для работы с реальными устройствами.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы Python и MQTT, понимать принципы работы умного дома и протоколов Zigbee/Z-Wave, уметь настраивать конфигурационные файлы YAML и работать с командной строкой Linux.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск Home Assistant
Для запуска Home Assistant выполнить команду hass или использовать systemctl: sudo systemctl start home-assistant.
3.2. Доступ к веб-интерфейсу
После запуска открыть браузер и перейти по адресу http://localhost:8123. При первом запуске необходимо создать учетную запись администратора.
3.3. Проверка работы MQTT брокера
Для проверки работы MQTT брокера выполнить: mosquitto_sub -t '#' -v

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Добавление устройства через MQTT
Для добавления устройства через MQTT необходимо создать конфигурацию в файле configuration.yaml:
mqtt:
  sensor:
    - name: "Температура"
      state_topic: "home/sensors/temperature"
  switch:
    - name: "Лампа"
      command_topic: "home/lights/living_room/set"
      state_topic: "home/lights/living_room/state"
4.2. Добавление Zigbee-устройств через Zigbee2MQTT
Для добавления Zigbee-устройств необходимо установить Zigbee2MQTT и настроить файл configuration.yaml:
mqtt:
  broker: localhost
  port: 1883
homeassistant:
  discovery: true
4.3. Создание автоматизации
Для создания автоматизации использовать интерфейс Automations или файл automations.yaml:
automation:
  - alias: "Включить свет при движении"
    trigger:
      platform: state
      entity_id: sensor.motion_sensor
      to: "on"
    action:
      service: switch.turn_on
      entity_id: switch.living_room_light
4.4. Создание сценария (Script)
Для создания сценария использовать файл scripts.yaml:
script:
  good_morning:
    sequence:
      - service: light.turn_on
        data:
          entity_id: light.bedroom
          brightness: 255
      - service: media_player.play_media
        data:
          entity_id: media_player.speaker
          media_content_id: "https://radio.ru/stream"
4.5. Добавление сенсоров (датчиков)
Для добавления датчиков в configuration.yaml:
sensor:
  - platform: mqtt
    name: "Влажность"
    state_topic: "home/sensors/humidity"
    unit_of_measurement: "%"
  - platform: mqtt
    name: "Освещенность"
    state_topic: "home/sensors/light"
    unit_of_measurement: "lx"
4.6. Настройка панели управления (Lovelace)
Для настройки панели управления перейти в Configuration -> Lovelace Dashboards. Нажать Add Dashboard и выбрать нужные карточки (Entities, Glance, Gauge, History Graph).
4.7. Настройка интеграций
Для настройки интеграций перейти в Configuration -> Integrations. Нажать Add Integration и выбрать нужную интеграцию (MQTT, Zigbee Home Automation, Z-Wave, Philips Hue).
4.8. Использование API для автоматизации
Для использования API Home Assistant использовать REST API:
curl -X POST -H "Authorization: Bearer YOUR_TOKEN" -H "Content-Type: application/json" -d '{"entity_id": "switch.living_room_light"}' http://localhost:8123/api/services/switch/turn_on

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Home Assistant не запускается необходимо проверить логи: sudo journalctl -u home-assistant -f. При ошибке MQTT connection failed необходимо проверить, что Mosquitto запущен: sudo systemctl status mosquitto. При ошибке Zigbee adapter not found необходимо проверить подключение USB-адаптера и права доступа: ls -l /dev/ttyUSB*. При ошибке Configuration invalid необходимо проверить синтаксис YAML-файлов с помощью валидатора. При ошибке Device unavailable необходимо проверить питание устройства и качество связи. При ошибке Database lock необходимо перезапустить Home Assistant или удалить файл home-assistant_v2.db-wal.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка темы оформления
Для настройки темы перейти в Configuration -> Themes. Выбрать тему (Light, Dark, Midnight) или создать собственную.
6.2. Настройка языка интерфейса
Для настройки языка перейти в Configuration -> General -> Language. Выбрать Russian.
6.3. Настройка панели управления (Lovelace)
Для настройки панели управления в режиме редактирования нажать Edit Dashboard. Добавить карточки: Entities (список устройств), Glance (быстрый просмотр), Gauge (показатели датчиков), History Graph (графики изменений).
6.4. Настройка уведомлений
Для настройки уведомлений перейти в Configuration -> Automations -> Add Automation. Выбрать триггер (например, изменение состояния) и действие (например, отправить push-уведомление).
6.5. Настройка голосового управления
Для настройки голосового управления перейти в Configuration -> Voice Assistants. Добавить голосового ассистента и настроить интеграцию с Vosk или Google Assistant.
6.6. Настройка резервного копирования
Для настройки резервного копирования перейти в Configuration -> Backup. Нажать Create Backup для создания резервной копии всей конфигурации.
