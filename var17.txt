ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с симулятором автомобильной CAN-шины «CANoe»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
CANoe — симулятор автомобильной CAN-шины, предназначенный для разработки, тестирования и отладки систем связи в автомобилях с использованием протоколов CAN, LIN, FlexRay и Ethernet.
1.2. Область применения
Используется разработчиками встроенного ПО для автомобильных информационно-развлекательных систем для симуляции CAN-шины и тестирования взаимодействия электронных блоков управления (ECU).
1.3. Краткая характеристика возможностей
CANoe позволяет симулировать CAN-сообщения, анализировать трафик CAN-шины, отлаживать работу электронных блоков управления, проверять соответствие требованиям ISO 26262 (функциональная безопасность), поддерживает работу с протоколами SOME/IP и DDS, предоставляет встроенный анализатор для мониторинга CAN-шины и поддерживает скриптовый язык CAPL для автоматизации тестирования.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы CANoe требуется операционная система Linux QNX или Windows, процессор архитектуры x86_64, оперативная память не менее 8 ГБ, свободное место на диске не менее 10 ГБ и наличие лицензионного ключа Vector.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы работы с CAN-протоколом, понимать принципы автомобильных сетей, уметь работать с сигналами и сообщениями CAN и знать язык CAPL для автоматизации.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск CANoe
Для запуска CANoe необходимо выполнить команду canoe или использовать ярлык на рабочем столе. При первом запуске необходимо активировать лицензионный ключ.
3.2. Создание нового проекта
Для создания нового проекта нажать File -> New Project. Выбрать тип проекта (CAN, LIN, FlexRay) и нажать OK.
3.3. Настройка CAN-интерфейса
Для настройки CAN-интерфейса перейти в Configuration -> Network Interfaces. Выбрать доступный CAN-интерфейс (виртуальный или физический).

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Добавление CAN-узлов
Для добавления CAN-узлов в симуляцию перейти в Simulation Setup. Нажать правой кнопкой мыши и выбрать Insert CAN Node. Ввести имя узла (например, ECU_Engine) и выбрать соответствующий DBC-файл.
4.2. Создание и отправка сообщений
Для создания сообщения в CAPL-скрипте использовать следующий код:
on start {
  message 0x123 msg;
  msg.dlc = 8;
  msg.byte(0) = 0x01;
  output(msg);
}
4.3. Мониторинг CAN-шины
Для мониторинга CAN-шины открыть Measurement Setup. Добавить блок Trace для визуализации сообщений. Нажать Start Measurement для начала мониторинга.
4.4. Загрузка DBC-файла
Для загрузки DBC-файла (базы данных CAN-сигналов) перейти в Simulation Setup -> Database. Выбрать Add Database и указать путь к .dbc файлу.
4.5. Использование CAPL для тестирования
Для создания тестового сценария на CAPL создать новый файл .can в папке Simulation. Пример теста:
void Test_EngineRPM() {
  message 0x100 msg;
  msg.dlc = 8;
  msg.byte(0) = 0x00;
  output(msg);
  testWaitForSignal("EngineRPM", 1000);
  testCheckSignal("EngineRPM", 0, 100);
}
4.6. Логирование трафика
Для логирования трафика CAN-шины перейти в Measurement Setup и добавить блок Logging. Указать имя файла для сохранения логов (.asc или .blf).
4.7. Просмотр логов
Для просмотра логов открыть File -> Open Log File и выбрать сохраненный файл .asc. Лог отобразится в окне Trace.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки License not found необходимо активировать лицензионный ключ через License Manager. При ошибке CAN interface not found необходимо проверить подключение CAN-интерфейса и настройки драйверов. При ошибке DBC file not found необходимо проверить путь к DBC-файлу или загрузить его заново. При ошибке CAPL compilation error необходимо проверить синтаксис CAPL-кода и наличие закрывающих скобок. При ошибке Bus-off необходимо проверить физическое подключение CAN-шины или отключить эхо-режим. При ошибке Timeout в тесте необходимо увеличить таймаут или проверить, что сообщения действительно отправляются.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка отображения в Trace
Для настройки отображения в Trace перейти в View -> Trace -> Configure. Настроить колонки для отображения: время, ID, DLC, данные.
6.2. Настройка цветов для CAN-сообщений
Для настройки цветов перейти в View -> Symbol Filter. Выбрать сообщение и установить цвет.
6.3. Настройка автоматического запуска тестов
Для настройки автоматического запуска тестов перейти в Test Module. Добавить CAPL-тесты и настроить автоматический запуск при старте симуляции.
6.4. Настройка сохранения логов
Для настройки автоматического сохранения логов перейти в Measurement Setup -> Logging. Выбрать режим и имя файла с датой.
6.5. Настройка графиков
Для добавления графиков перейти в Graphics. Добавить сигналы для отображения в реальном времени (скорость, обороты двигателя).
