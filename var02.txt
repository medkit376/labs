ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе со средой разработки «Android Studio»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Android Studio — официальная интегрированная среда разработки (IDE) для создания мобильных приложений на платформах Android и iOS с использованием фреймворка Flutter и языка Dart.
1.2. Область применения
Используется разработчиками мобильных приложений для создания кроссплатформенных приложений из единой кодовой базы.
1.3. Краткая характеристика возможностей
Android Studio позволяет создавать проекты на Flutter, редактировать код на Dart с автодополнением, запускать приложения на эмуляторах Android и реальных устройствах, визуально проектировать интерфейсы с помощью Flutter Inspector, управлять зависимостями через pubspec.yaml и отлаживать приложения с горячей перезагрузкой (Hot Reload).

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Android Studio требуется операционная система Ubuntu 20.04 и выше, оперативная память не менее 8 ГБ, процессор архитектуры x86_64 с поддержкой виртуализации, свободное место на диске не менее 10 ГБ и графический адаптер с поддержкой OpenGL 2.0.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы языка Dart, принципы работы Flutter-фреймворка, базовые понятия мобильной разработки (Activity, Widget, State) и уметь работать с эмуляторами мобильных устройств.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск программы
Для запуска Android Studio необходимо открыть терминал и выполнить команду /opt/android-studio/bin/studio.sh или использовать иконку в меню приложений.
3.2. Создание нового проекта
После запуска Android Studio на экране приветствия необходимо выбрать New Project. Затем выбрать шаблон Flutter -> Application. Указать путь к Flutter SDK (если не найден автоматически) и задать имя проекта, например, my_first_app. Нажать Finish.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Структура проекта
В панели Project отображается структура проекта. Основные файлы и папки: lib/main.dart — главный файл с кодом приложения; pubspec.yaml — файл манифеста для управления зависимостями (библиотеками); android/ и ios/ — папки с платформенно-зависимым кодом.
4.2. Редактирование кода
Для редактирования кода необходимо открыть файл lib/main.dart. Android Studio поддерживает автодополнение, подсветку синтаксиса, рефакторинг и навигацию по коду. Для сохранения изменений используется сочетание клавиш Ctrl+S.
4.3. Визуальное проектирование интерфейсов (Flutter Inspector)
Для открытия Flutter Inspector необходимо запустить приложение на эмуляторе или устройстве, затем нажать на иконку Flutter Inspector на правой панели (или через View -> Tool Windows -> Flutter Inspector). В окне инспектора отображается дерево виджетов (Widget Tree). При выделении виджета в дереве подсвечивается соответствующий элемент на экране устройства. Свойства виджета (цвет, размер, отступы) можно изменять и наблюдать изменения в реальном времени после нажатия Hot Reload (иконка молнии).
4.4. Запуск на эмуляторе
Для запуска приложения необходимо в верхней панели выбрать устройство из выпадающего списка (например, Pixel 6 API 33). Затем нажать зеленую кнопку Run (▶) или использовать сочетание клавиш Shift+F10. Приложение скомпилируется и установится на выбранный эмулятор или реальное устройство.
4.5. Создание эмулятора (AVD)
Для создания нового эмулятора необходимо открыть Device Manager (иконка телефона на панели инструментов). Нажать Create device. Выбрать устройство (например, Pixel 6) и нажать Next. Выбрать образ системы (например, Tiramisu API 33) и скачать его, если отсутствует. Нажать Next, задать имя эмулятора и нажать Finish.
4.6. Тестирование на разных устройствах
В Device Manager можно создавать несколько эмуляторов с разными версиями ОС и разрешениями экранов. Для запуска приложения на другом устройстве необходимо остановить текущий сеанс, выбрать другое устройство из выпадающего списка и нажать Run.
4.7. Управление зависимостями
Для добавления библиотеки необходимо открыть файл pubspec.yaml. В разделе dependencies добавить новую зависимость, например: http: ^0.13.0. Затем нажать Pub get в верхней части файла или выполнить в терминале команду flutter pub get.
4.8. Отладка приложения
Для отладки необходимо установить точки останова (Breakpoints) в коде, нажав на левую область напротив строки. Затем запустить приложение в режиме Debug (кнопка с жучком или Shift+F9). При достижении точки останова выполнение приостанавливается, и можно просматривать значения переменных в панели Debug.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Flutter SDK not found необходимо указать путь к Flutter SDK в настройках: File -> Settings -> Languages & Frameworks -> Flutter -> Flutter SDK path. При ошибке No connected devices необходимо запустить эмулятор через AVD Manager или подключить реальное устройство. При ошибке Gradle sync failed следует проверить интернет-соединение и настройки прокси. При ошибке Hot Reload не работает необходимо перезапустить приложение или выполнить flutter clean в терминале. При ошибке Ошибка компиляции необходимо проверить вывод вкладки Run или Build, обратить внимание на отступы в коде Dart (чувствителен к пробелам). При зависании эмулятора необходимо перезапустить эмулятор через AVD Manager (нажать иконку сброса питания).

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Смена темы оформления
Для смены темы оформления необходимо нажать File -> Settings -> Appearance & Behavior -> Appearance -> Theme и выбрать тему IntelliJ Light, Darcula или High contrast.
6.2. Настройка размера шрифта
Для изменения размера шрифта редактора необходимо нажать File -> Settings -> Editor -> Font и изменить значение Size. Для изменения размера шрифта интерфейса необходимо нажать File -> Settings -> Appearance & Behavior -> Appearance -> Size.
6.3. Настройка масштабирования для HiDPI
Для настройки масштабирования необходимо открыть Help -> Edit Custom VM Options и добавить строку -Dsun.java2d.uiScale=1.5. Затем перезапустить Android Studio.
6.4. Настройка клавиатурных сокращений
Для настройки клавиатурных сокращений необходимо нажать File -> Settings -> Keymap. Выбрать схему (например, Eclipse) или настроить отдельные сочетания клавиш.
6.5. Настройка плагинов
Для установки плагинов необходимо нажать File -> Settings -> Plugins. Ввести название плагина в строке поиска и нажать Install. Рекомендуется установить плагины: Flutter, Dart, Flutter Intl, Rainbow Brackets.
6.6. Включение панели инструментов
Для включения панели инструментов необходимо нажать View -> Appearance -> Toolbar.
