ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с системой резервного копирования и восстановления «Bacula»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Bacula — система резервного копирования и восстановления данных, предназначенная для автоматического создания резервных копий серверов и восстановления данных в случае сбоя.
1.2. Область применения
Используется администраторами серверной инфраструктуры для резервного копирования критически важных данных и обеспечения возможности восстановления после аппаратных или программных сбоев.
1.3. Краткая характеристика возможностей
Bacula позволяет создавать полные, дифференциальные и инкрементные резервные копии, восстанавливать файлы и целые каталоги, управлять расписанием резервного копирования, хранить бэкапы на различных носителях (диски, ленты, сетевые хранилища), поддерживает шифрование и сжатие данных, предоставляет веб-интерфейс (Bacula Web) и интерфейс командной строки для управления.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Bacula требуется операционная система Ubuntu 20.04 и выше, оперативная память не менее 2 ГБ, процессор архитектуры x86_64, свободное место на диске для хранения бэкапов (зависит от объема данных) и установленный PostgreSQL или MySQL для хранения каталога.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы администрирования Linux, понимать структуру файловой системы, уметь работать с командной строкой и знать базовые принципы резервного копирования.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Проверка установки Bacula
Для проверки установки Bacula необходимо выполнить команду bacula-version. При успешной установке отобразится версия Bacula.
3.2. Настройка конфигурации Bacula
Основные конфигурационные файлы Bacula находятся в /etc/bacula/: bacula-dir.conf — конфигурация директора; bacula-fd.conf — конфигурация файлового демона; bacula-sd.conf — конфигурация демона хранения; bconsole.conf — конфигурация консоли.
3.3. Запуск Bacula
Для запуска Bacula необходимо выполнить: sudo systemctl start bacula-director, sudo systemctl start bacula-fd, sudo systemctl start bacula-sd.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Запуск консоли Bacula
Для запуска консоли Bacula используется команда: bconsole. После запуска отобразится приглашение *. Для выхода из консоли использовать команду quit.
4.2. Проверка статуса Bacula
Для проверки статуса всех компонентов Bacula в консоли выполнить команду: status. Для проверки статуса директора: status director. Для проверки статуса демона хранения: status storage. Для проверки статуса файлового демона: status client.
4.3. Запуск резервного копирования вручную
Для ручного запуска резервного копирования в консоли выполнить команду: run. Выбрать клиент (client), набор файлов (fileset) и пул хранения (storage). Подтвердить запуск.
4.4. Просмотр каталога бэкапов
Для просмотра каталога бэкапов в консоли выполнить команду: list files jobid=JOB_ID. Для поиска файла в каталоге: list files jobid=JOB_ID pattern=/path/to/file.
4.5. Восстановление файлов
Для восстановления файлов в консоли выполнить команду: restore. Выбрать клиент и способ восстановления (по времени, по последнему бэкапу). Выбрать файлы для восстановления. Указать место восстановления.
4.6. Просмотр всех заданий
Для просмотра всех заданий в консоли выполнить команду: list jobs. Для просмотра заданий с фильтром по статусу: list jobs status=Failed.
4.7. Отмена задания
Для отмены выполнения задания в консоли выполнить команду: cancel jobid=JOB_ID.
4.8. Настройка расписания
Для настройки расписания необходимо отредактировать файл /etc/bacula/bacula-dir.conf. Добавить Schedule:
Schedule {
  Name = Weekly
  Run = Full every 7 days at 02:00
  Run = Differential every 1 day at 02:00
}
4.9. Просмотр логов Bacula
Логи Bacula находятся в /var/log/bacula/. Основные файлы: director.log — логи директора; fd.log — логи файлового демона; sd.log — логи демона хранения.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Bacula director not running необходимо запустить директор: sudo systemctl start bacula-director. При ошибке Connection refused при подключении к демону хранения необходимо проверить статус bacula-sd и порт (по умолчанию 9102). При ошибке No available storage device необходимо проверить доступность устройства хранения и права на запись. При ошибке Job cancelled необходимо проверить, что задание не было отменено вручную. При ошибке Full backup failed необходимо проверить наличие свободного места на устройстве хранения. При ошибке Restore failed необходимо проверить, что файлы существуют в каталоге бэкапов.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка Bacula Web
Для настройки Bacula Web необходимо установить пакет: sudo apt install -y bacula-web. Настроить подключение к базе данных Bacula в /etc/bacula-web/config.php.
6.2. Настройка уведомлений по email
Для настройки уведомлений необходимо в файле bacula-dir.conf добавить параметр Mail в Job:
Job {
  Name = BackupJob
  Mail = admin@company.com
  MailOnSuccess = yes
  MailOnFailure = yes
}
6.3. Настройка шифрования бэкапов
Для настройки шифрования необходимо в файле bacula-sd.conf добавить параметр Encryption:
Storage {
  Name = MyStorage
  Encryption = yes
  EncryptionKey = "key"
}
6.4. Настройка сжатия бэкапов
Для настройки сжатия необходимо в файле bacula-dir.conf добавить параметр Compression:
Job {
  Name = BackupJob
  Compression = GZIP
}
6.5. Настройка очистки старых бэкапов
Для настройки очистки старых бэкапов необходимо в файле bacula-dir.conf добавить параметр Retention:
Pool {
  Name = Default
  Volume Retention = 30 days
  Job Retention = 30 days
  File Retention = 60 days
}
6.6. Настройка количества параллельных заданий
Для настройки параллельности заданий в bacula-dir.conf:
Director {
  Name = MyDirector
  MaxConcurrentJobs = 10
}
