ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с системой визуализации энергосетей «Grafana + Prometheus»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Grafana + Prometheus — система мониторинга и визуализации для сбора данных с энергооборудования, анализа больших потоков телеметрии и моделирования режимов работы энергосистем в реальном времени.
1.2. Область применения
Используется разработчиками SCADA-систем для энергетики для визуализации энергосетей, анализа потоков телеметрии, прогнозирования режимов работы энергосистем и обеспечения защиты критической инфраструктуры.
1.3. Краткая характеристика возможностей
Grafana позволяет создавать панели мониторинга энергосетей, строить графики потоков данных, настраивать аварийные оповещения, интегрироваться с InfluxDB и другими источниками данных, поддерживать SSL/TLS для защищенных соединений и работать в режиме реального времени. Prometheus собирает метрики с энергооборудования, хранит данные с высокой точностью и поддерживает API для запросов к данным.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Grafana + Prometheus требуется операционная система Ubuntu 20.04 и выше, процессор архитектуры x86_64, оперативная память не менее 8 ГБ, свободное место на диске не менее 20 ГБ и сетевое соединение с энергооборудованием.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы работы с промышленными протоколами (IEC 61850, DNP3), понимать принципы сбора и анализа телеметрии, уметь работать с панелями визуализации и знать язык запросов PromQL.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск Grafana
Для запуска Grafana выполнить sudo systemctl start grafana-server. Открыть браузер и перейти по адресу http://localhost:3000. Войти с учетными данными по умолчанию (admin/admin).
3.2. Запуск Prometheus
Для запуска Prometheus выполнить /opt/prometheus/prometheus. Конфигурационный файл prometheus.yml находится в /opt/prometheus/.
3.3. Добавление источника данных
В Grafana перейти в Configuration -> Data Sources. Нажать Add data source и выбрать Prometheus. Указать URL: http://localhost:9090. Нажать Save & Test.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Сбор метрик с энергооборудования
Для сбора метрик настроить Prometheus для scraping данных с энергооборудования через экспортеры (Modbus, IEC 61850, DNP3). Добавить job в prometheus.yml:
scrape_configs:
  - job_name: 'energy'
    static_configs:
      - targets: ['192.168.1.100:502']  # Modbus device
4.2. Создание панели мониторинга
В Grafana нажать + -> Create Dashboard. Нажать Add new panel. Выбрать тип графика (Graph, Stat, Gauge). Ввести запрос PromQL: energy_power_total или rate(energy_consumption[1h]).
4.3. Визуализация потоков энергии
Для визуализации потоков энергии использовать панель "Gauge" или "Stat". Настроить отображение текущей мощности, напряжения, частоты, коэффициента мощности. Добавить цветовые индикаторы для критических значений.
4.4. Создание трендов нагрузки
Для создания трендов нагрузки использовать панель "Graph". Ввести запрос: energy_consumption_sum. Настроить агрегацию (avg, max, min) и периоды (1h, 24h, 7d, 30d).
4.5. Настройка аварийных оповещений
Для настройки оповещений перейти в Alerting. Нажать Create Alert. Выбрать источник данных Prometheus. Настроить условие: when energy_power_total > 1000 for 5m. Настроить уведомления (Email, Telegram, Slack).
4.6. Прогнозирование режимов работы
Для прогнозирования использовать панель "Graph" с предсказанием. Применить функцию predict_linear(energy_consumption[1h], 6h) для прогноза на 6 часов.
4.7. Анализ больших потоков телеметрии
Для анализа больших потоков использовать панель "Table". Вывести данные за период: energy_consumption_total[30d]. Экспортировать данные в CSV для дальнейшего анализа.
4.8. Интеграция с InfluxDB для долговременного хранения
Для интеграции с InfluxDB добавить источник данных в Grafana (InfluxDB). Настроить подключение: URL http://localhost:8086, Database energy_data.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Prometheus connection failed необходимо проверить, что Prometheus запущен и доступен на порту 9090. При ошибки Data source not found необходимо добавить источник данных в Grafana. При ошибке No data points необходимо проверить запрос PromQL и наличие метрик в Prometheus. При ошибке Dashboard не отображается необходимо проверить права доступа и наличие панели. При ошибке Alert rule failed необходимо проверить синтаксис условия и наличие данных. При ошибке SSL certificate error необходимо настроить корректные сертификаты в Grafana.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка единиц измерения
Для настройки единиц измерения в Grafana перейти в Panel -> Field -> Unit. Выбрать единицы: Power (W, kW, MW), Energy (J, Wh, kWh), Voltage (V), Current (A), Frequency (Hz).
6.2. Настройка цветовой схемы для критических значений
Для настройки цветовой схемы использовать Thresholds в панели. Установить значения для нормального состояния (зеленый), предупреждения (желтый), критического (красный).
6.3. Настройка автоматического обновления панелей
Для настройки автоматического обновления перейти в Dashboard -> Settings -> Time and Refresh. Настроить интервал обновления (5s, 10s, 30s, 1m).
6.4. Настройка пользователей и прав доступа
Для настройки пользователей перейти в Configuration -> Users. Создать пользователей с ролями: Viewer (только чтение), Editor (может редактировать панели), Admin (полный доступ).
6.5. Настройка SSL/TLS для защищенного соединения
Для настройки SSL/TLS отредактировать файл /etc/grafana/grafana.ini:
[server]
protocol = https
cert_file = /etc/ssl/certs/grafana.crt
cert_key = /etc/ssl/private/grafana.key
6.6. Настройка автоматического создания отчетов
Для настройки автоматического создания отчетов перейти в Reporting. Настроить расписание (ежедневно, еженедельно, ежемесячно). Выбрать формат (PDF, PNG). Настроить отправку по email.
