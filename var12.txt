ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с SDK для AR/VR шлемов «OpenXR»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
OpenXR — открытый стандарт для разработки приложений дополненной и виртуальной реальности, обеспечивающий единый API для работы с различными AR/VR шлемами (Oculus, HTC Vive, HoloLens, Valve Index).
1.2. Область применения
Используется разработчиками AR/VR приложений для создания универсальных приложений, работающих на различных устройствах без изменения кода.
1.3. Краткая характеристика возможностей
OpenXR позволяет подключаться к различным AR/VR устройствам через единый API, отслеживать позицию головы и контроллеров (6DoF), обрабатывать ввод с контроллеров, поддерживать стереоскопический рендеринг, управлять пространственными привязками (spatial anchors) и обеспечивать высокую производительность в реальном времени.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы OpenXR требуется операционная система Ubuntu 20.04 и выше, установленный игровой движок (Unity), драйверы для VR/AR шлема, оперативная память не менее 8 ГБ, процессор архитектуры x86_64 и GPU с поддержкой Vulkan или OpenGL 4.6.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы разработки на C# (для Unity), понимать принципы 3D-графики, знать основы работы с AR/VR устройствами и уметь работать с Unity и его Package Manager.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Установка OpenXR в Unity
Для установки OpenXR в Unity необходимо открыть Package Manager (Window -> Package Manager). Переключиться на Packages: Unity Registry. Найти пакет OpenXR Plugin и установить его.
3.2. Настройка OpenXR в проекте
После установки OpenXR Plugin необходимо перейти в Project Settings -> Player -> XR Settings. Включить OpenXR в списке плагинов. Нажать на иконку шестеренки рядом с OpenXR и выбрать свойств.
3.3. Добавление устройств в OpenXR
В настройках OpenXR необходимо добавить поддерживаемые устройства. Нажать на + и выбрать Oculus Touch, HTC Vive, Windows Mixed Reality или другие.
3.4. Настройка взаимодействия (Interactions)
Для настройки взаимодействия необходимо добавить Interaction Profile для каждого контроллера. Выбрать профиль (Oculus Touch, Valve Index и т.д.) и настроить привязки действий (Actions).

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Создание XR-сцены
Для создания XR-сцены необходимо создать новую сцену в Unity. Добавить GameObject -> XR -> XR Origin (или XR Rig). Это будет корневой объект для камеры и контроллеров.
4.2. Настройка XR Origin
В компоненте XR Origin настроить следующие параметры: Camera Y Offset (высота камеры), Tracking Origin Mode (Floor или Device). Добавить подкомпоненты: Locomotion System (для телепортации), Continuous Move Provider (для плавного перемещения).
4.3. Добавление взаимодействия с объектами
Для добавления взаимодействия необходимо добавить компонент XR Interaction Manager к XR Origin. Для объектов, с которыми можно взаимодействовать, добавить компонент XR Grab Interactable. Для контроллера добавить компонент XR Direct Interactor.
4.4. Отслеживание позиции головы
Отслеживание позиции головы происходит автоматически через XR Origin. Для получения данных о позиции и повороте головы в коде: Camera.main.transform.position и Camera.main.transform.rotation.
4.5. Обработка ввода с контроллеров
Для обработки ввода с контроллеров необходимо создать Action Maps через Input System. Пример для получения нажатия кнопки:
private void Update() {
    var device = InputDevices.GetDeviceAtXRNode(XRNode.RightHand);
    if (device.TryGetFeatureValue(CommonUsages.triggerButton, out bool pressed) && pressed) {
        // Действие при нажатии триггера
    }
}
4.6. Тестирование в редакторе Unity
Для тестирования в редакторе без шлема необходимо включить Simulator: Window -> XR -> XR Simulation. Это позволит управлять камерой с помощью мыши и клавиатуры.
4.7. Сборка приложения
Для сборки приложения необходимо в Build Settings выбрать платформу (Windows, Android, iOS). Затем нажать Build и выбрать папку для выходных файлов.
4.8. Тестирование на реальном устройстве
Для тестирования на реальном устройстве необходимо подключить шлем к компьютеру, запустить сборку и установить приложение на устройство. Для Oculus Quest необходимо включить режим разработчика и использовать ADB для установки.

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки OpenXR Plugin not found необходимо переустановить пакет через Package Manager. При ошибке Device not detected необходимо проверить подключение шлема и установку драйверов. При ошибке Action not bound необходимо настроить привязки действий в настройках OpenXR. При ошибке XR Origin not set необходимо добавить XR Origin в сцену. При ошибке Performance issues необходимо уменьшить графические настройки и проверить оптимизацию кода. При ошибке Simulator не реагирует необходимо проверить настройки XR Simulation в Window -> XR -> XR Simulation.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка OpenXR в Project Settings
Для настройки OpenXR перейти в Edit -> Project Settings -> XR Plugin Management -> OpenXR. В этом меню доступны: выбор устройств, настройка взаимодействий, настройка отслеживания.
6.2. Настройка Input System для OpenXR
Для настройки Input System необходимо открыть Window -> Input System -> Input Actions. Создать новый Action Map для XR-контроллеров. Добавить действия: LeftHand, RightHand, Head.
6.3. Настройка Render Scale
Для изменения разрешения рендеринга в OpenXR: XRSettings.eyeTextureResolutionScale = 1.5f; (значение 1.0 - стандартное разрешение, 1.5 - повышенное).
6.4. Настройка антиалиасинга
Для настройки антиалиасинга в OpenXR: QualitySettings.antiAliasing = 8; (уровень 2, 4, 8). В OpenXR также можно включить MSAA в настройках Project Settings.
6.5. Настройка отслеживания Floor или Device
Для выбора режима отслеживания в XR Origin: Tracking Origin Mode -> Floor (пользователь стоит на полу) или Device (отслеживание от устройства).
6.6. Настройка слоев для производительности
Для повышения производительности используйте Render Layers для разделения объектов по слоям рендеринга. Это уменьшает количество отрисовываемых объектов.
