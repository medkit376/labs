ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с платформой CI/CD «Jenkins»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Jenkins — открытая платформа для автоматизации сборки, тестирования и развертывания приложений (CI/CD), предназначенная для непрерывной интеграции и доставки программного обеспечения.
1.2. Область применения
Используется DevOps-инженерами для автоматизации процессов разработки, сборки, тестирования и развертывания приложений в различных средах.
1.3. Краткая характеристика возможностей
Jenkins позволяет создавать и запускать пайплайны сборки, автоматически выполнять тестирование при каждом изменении кода, развертывать приложения на серверах и в контейнерах, интегрироваться с системами контроля версий (Git, SVN), поддерживает тысячи плагинов для расширения функциональности и предоставляет веб-интерфейс для управления проектами.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Jenkins требуется операционная система Ubuntu 20.04 и выше, оперативная память не менее 4 ГБ, процессор архитектуры x86_64, свободное место на диске не менее 10 ГБ, установленный Java (JDK 11 или 17) и доступ к сети для загрузки плагинов и взаимодействия с Git-репозиториями.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы работы с системами контроля версий (Git), понимать принципы CI/CD, уметь работать с веб-интерфейсом Jenkins, знать основы написания скриптов (Bash, Groovy) и понимать принципы контейнеризации (Docker).

3. ПОДГОТОВКА К РАБОТЕ
3.1. Запуск Jenkins
Для запуска Jenkins необходимо убедиться, что служба запущена: sudo systemctl status jenkins. Если служба не запущена, выполнить sudo systemctl start jenkins. Jenkins по умолчанию работает на порту 8080. Открыть браузер и перейти по адресу: http://localhost:8080
3.2. Получение пароля администратора
При первом запуске Jenkins запрашивает пароль администратора. Получить пароль можно командой: sudo cat /var/lib/jenkins/secrets/initialAdminPassword. Скопировать пароль и вставить его в веб-интерфейсе.
3.3. Установка рекомендуемых плагинов
После входа в систему Jenkins предложит установить плагины. Выбрать «Install suggested plugins» для установки рекомендуемого набора плагинов, необходимых для базовой работы.
3.4. Создание администратора
После установки плагинов необходимо создать учетную запись администратора: ввести имя пользователя, пароль, полное имя и email. Запомнить или записать учетные данные.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Создание нового проекта
Для создания нового проекта необходимо нажать «New Item» на главной странице Jenkins. Ввести имя проекта. Выбрать тип проекта (Freestyle project — для простых задач, Pipeline — для сложных пайплайнов). Нажать «OK».
4.2. Настройка привязки к Git-репозиторию
В настройках проекта перейти в раздел «Source Code Management». Выбрать «Git». Ввести URL репозитория (например, https://github.com/user/project.git). При необходимости указать учетные данные для доступа к приватному репозиторию.
4.3. Настройка триггеров сборки
Для автоматического запуска сборки при изменении кода в разделе «Build Triggers» выбрать «Poll SCM» и указать расписание в формате cron (например, H/5 * * * * — проверять каждые 5 минут). Либо выбрать «GitHub hook trigger for GITScm polling» для интеграции с GitHub.
4.4. Настройка шагов сборки (Build Steps)
В разделе «Build Steps» нажать «Add build step» и выбрать необходимое действие. Для выполнения команд в Shell: выбрать «Execute shell» и ввести команды (например, ./gradlew build для сборки Java-проекта или docker build -t myapp . для сборки Docker-образа).
4.5. Настройка шагов после сборки (Post-build Actions)
В разделе «Post-build Actions» нажать «Add post-build action». Для отправки уведомлений по email выбрать «Editable Email Notification». Для архивации артефактов выбрать «Archive the artifacts» и указать путь к файлам (например, build/*.jar).
4.6. Запуск сборки вручную
Для ручного запуска сборки необходимо открыть проект и нажать кнопку «Build Now». В разделе «Build History» отобразится статус текущей сборки и история всех сборок.
4.7. Просмотр логов сборки
Для просмотра логов сборки необходимо нажать на номер сборки в разделе «Build History», затем выбрать «Console Output». В окне отобразится полный вывод команд сборки, включая ошибки.
4.8. Создание Pipeline (Декларативный)
Для создания декларативного Pipeline необходимо в настройках проекта выбрать тип «Pipeline». В разделе «Pipeline» выбрать «Pipeline script» или «Pipeline script from SCM». Пример скрипта:
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                sh 'make'
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                sh 'make test'
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                sh 'make deploy'
            }
        }
    }
}
4.9. Управление плагинами
Для установки новых плагинов необходимо перейти в «Manage Jenkins» -> «Manage Plugins» -> «Available». Найти нужный плагин, отметить его и нажать «Install without restart». Для обновления плагинов перейти во вкладку «Updates» и нажать «Download now and install after restart».
4.10. Настройка учетных данных
Для добавления учетных данных необходимо перейти в «Manage Jenkins» -> «Manage Credentials» -> «Global» -> «Add Credentials». Выбрать тип (Username with password, SSH Username with private key, Secret text). Ввести учетные данные и нажать «Create».

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Jenkins не запускается необходимо проверить статус службы командой sudo systemctl status jenkins и просмотреть логи в /var/log/jenkins/jenkins.log. При ошибке Build failed необходимо проверить логи сборки в разделе Console Output, исправить ошибки в коде или конфигурации сборки. При ошибке Connection refused при подключении к Git необходимо проверить URL репозитория и учетные данные. При ошибке No such plugin необходимо установить недостающий плагин через Manage Plugins. При ошибке OutOfMemoryError необходимо увеличить объем оперативной памяти для Jenkins: отредактировать файл /etc/default/jenkins и изменить параметр JAVA_ARGS на -Xmx2048m. При ошибке 403 Forbidden необходимо проверить права доступа и учетные данные пользователя. При ошибке Timeout необходимо увеличить таймаут в настройках сборки или оптимизировать процесс сборки.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Смена темы оформления
Для смены темы необходимо установить плагин «Simple Theme Plugin» через Manage Plugins. Затем перейти в «Manage Jenkins» -> «Configure System» -> «Theme» и выбрать тему или загрузить собственный CSS-файл.
6.2. Настройка отображения dashboard
Для изменения отображения главной страницы необходимо установить плагин «Dashboard View». Затем создать новый вид с необходимыми фильтрами и колонками.
6.3. Настройка уведомлений по email
Для настройки email-уведомлений необходимо перейти в «Manage Jenkins» -> «Configure System» -> «E-mail Notification». Ввести SMTP-сервер, порт, учетные данные и email отправителя. В настройках проекта в разделе «Post-build Actions» добавить «Editable Email Notification» и настроить получателей.
6.4. Настройка русского языка в интерфейсе
Для установки русского языка необходимо установить плагин «Locale Plugin». Затем перейти в «Manage Jenkins» -> «Configure System» -> «Locale» и ввести значение «ru».
6.5. Настройка Jenkins URL
Для настройки URL Jenkins необходимо перейти в «Manage Jenkins» -> «Configure System» -> «Jenkins Location» и ввести полный URL (например, http://jenkins.company.com:8080/).
6.6. Настройка количества сохраняемых сборок
Для ограничения количества хранимых сборок необходимо перейти в настройки проекта. В разделе «Discard old builds» указать максимальное количество сборок (например, 50) и срок хранения в днях (например, 30). Для глобальной настройки перейти в «Manage Jenkins» -> «Configure System» -> «Discard old builds».
