ДОКУМЕНТАЦИЯ: РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование документа: Руководство пользователя по работе с системой нагрузочного тестирования «Locust»
Версия: 1.0
Дата: 30.06.2026
Разработчик: Техник-программист

1. ВВЕДЕНИЕ
1.1. Назначение программы
Locust — система нагрузочного тестирования, предназначенная для симуляции тысяч виртуальных игроков, выполняющих действия в многопользовательских онлайн-играх, с целью проверки производительности и стабильности серверов.
1.2. Область применения
Используется разработчиками онлайн-игр для тестирования серверной логики, проверки устойчивости к нагрузкам, выявления узких мест и оптимизации производительности игровых серверов.
1.3. Краткая характеристика возможностей
Locust позволяет симулировать поведение тысяч виртуальных игроков (пользователей), выполнять различные сценарии (авторизация, игровые действия, чат), измерять время ответа сервера, отслеживать количество ошибок, предоставлять веб-интерфейс для управления тестом в реальном времени и генерировать отчеты о производительности.

2. ТРЕБОВАНИЯ К УСЛОВИЯМ ЭКСПЛУАТАЦИИ
2.1. Минимальные системные требования
Для работы Locust требуется операционная система Ubuntu 20.04 и выше, установленный Python 3.8 и выше, оперативная память не менее 4 ГБ, процессор архитектуры x86_64 и свободное место на диске не менее 1 ГБ.
2.2. Требования к квалификации пользователя
Пользователь должен знать основы Python, понимать архитектуру клиент-серверных приложений, уметь писать сценарии нагрузочного тестирования и анализировать результаты тестов.

3. ПОДГОТОВКА К РАБОТЕ
3.1. Проверка установки
Для проверки установки Locust выполнить команду locust --version. При успешной установке отобразится версия Locust.
3.2. Создание файла сценария
Для создания сценария нагрузочного тестирования создать файл locustfile.py. Базовый сценарий включает импорт библиотек и определение класса пользователя.

4. ОПИСАНИЕ ОПЕРАЦИЙ
4.1. Базовый сценарий для игры
Для создания базового сценария игрока:
from locust import HttpUser, task, between

class GamePlayer(HttpUser):
    wait_time = between(1, 3)

    def on_start(self):
        self.client.post("/api/login", json={"username": "player1", "password": "pass123"})

    @task(3)
    def join_game(self):
        self.client.get("/api/game/join")

    @task(1)
    def make_move(self):
        self.client.post("/api/game/move", json={"x": 10, "y": 20})

    @task(1)
    def chat(self):
        self.client.post("/api/chat", json={"message": "Hello!"})
4.2. Запуск Locust в веб-режиме
Для запуска Locust с веб-интерфейсом выполнить: locust -f locustfile.py --host http://game-server.com
Открыть браузер и перейти по адресу http://localhost:8089
4.3. Запуск Locust в головном режиме (без веб-интерфейса)
Для запуска Locust в головном режиме выполнить: locust -f locustfile.py --headless -u 1000 -r 100 --host http://game-server.com
4.4. Настройка количества пользователей
В веб-интерфейсе установить параметры: Number of users (количество виртуальных игроков), Spawn rate (скорость добавления игроков в секунду). Нажать Start Swarming.
4.5. Просмотр статистики в реальном времени
В веб-интерфейсе доступна статистика: Requests (количество запросов), Fails (количество ошибок), Response Time (время ответа), RPS (Requests Per Second). Также доступны графики изменения нагрузки и времени отклика.
4.6. Анализ ошибок
Для просмотра ошибок перейти на вкладку Failures. Здесь отображаются все ошибки с указанием URL и кода ответа.
4.7. Генерация отчета
Для генерации отчета в головном режиме использовать параметр --html: locust -f locustfile.py --headless -u 1000 -r 100 --host http://game-server.com --html report.html
4.8. Использование переменных окружения
Для использования переменных окружения в сценарии:
import os
HOST = os.environ.get("GAME_HOST", "http://localhost:8080")
self.client.post(f"{HOST}/api/login", json={"username": "player1"})

5. АВАРИЙНЫЕ СИТУАЦИИ
При возникновении ошибки Connection refused необходимо проверить, что тестируемый сервер запущен и доступен по указанному адресу. При ошибке Too many connections необходимо увеличить лимит открытых файлов: ulimit -n 65535. При ошибке Timeout при выполнении запроса необходимо уменьшить количество одновременных пользователей или увеличить таймаут. При ошибке No module named 'locust' необходимо установить Locust: pip3 install locust. При ошибке Worker failed при распределенном тестировании необходимо проверить, что все worker-процессы запущены. При ошибке Out of memory необходимо уменьшить количество пользователей или использовать распределенный режим.

6. НАСТРОЙКА ИНТЕРФЕЙСА
6.1. Настройка веб-интерфейса
Для настройки веб-интерфейса использовать параметры: --web-port (порт веб-интерфейса), --web-host (хост для веб-интерфейса).
6.2. Настройка распределенного тестирования
Для настройки распределенного тестирования запустить мастер: locust -f locustfile.py --master
Запустить worker: locust -f locustfile.py --worker --master-host=192.168.1.100
6.3. Настройка таймаута запросов
Для настройки таймаута в сценарии использовать параметр:
self.client.post("/api/game/move", json={"x": 10}, timeout=5.0)
6.4. Настройка тегов для фильтрации задач
Для настройки тегов в сценарии:
@task(3)
@tag("critical")
def join_game(self):
    self.client.get("/api/game/join")
Запуск только с тегом critical: locust -f locustfile.py -T critical
6.5. Настройка логирования
Для настройки логирования использовать параметр: --logfile locust.log
Для подробного логирования использовать: --loglevel DEBUG
6.6. Настройка графиков в веб-интерфейсе
В веб-интерфейсе доступны вкладки: Charts (графики), History (история тестов), CSV (экспорт данных в CSV). Для настройки отображения использовать фильтры по URL.
